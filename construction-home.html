Reilly Contracting Website


Objective Wireframes Mockups Prototype
For this project, the client wanted a website that embodied their business in a way that differentiated them from other businesses. This construction company wanted a website that did not have the typical industrial theme, but rather a theme that portrayed friendliness, neighborhood, and the beach.

Objective:
to create a website that portrays a small construction company that is located in a private beach town where everyone knows each other.

Information Architecture

There wasn't much wireframing for this project, since the client was relatively lax about the layout for the website and wanted it to be very simple. The business didn't have enough material yet for a multi-page site, so I suggested that a one page layout would be the best fit.



Wireframes - Round 1

The first round of wireframes started with the one page layout. When the client reviewed the wireframe, they were initially pleased but did have some desired changes.

The client wanted to change around the style of the navigation menu, as they didn't want their logo to be the main attraction of the site. We also decided that the main image on the home page would span the entire page. The client also wanted to redo the portfolio section so that only two of the projects would be in the view as opposed to the three, just so that the projects would be bigger and more "clickable".


Wireframes - Round 2

Moving onto the second round, there were some minor changes that were made were to the navigation menu, portfolio, and images.

In this round, the navigation menu is pinned to the top of the page and fixed so that it moves with the page while the user is scrolling. It is more of a standard navigation bar as opposed to the prior one, so it was a better choice as the audience for this website ranges from very tech-literate to low tech-literate.

However, from this wireframe, the inclusion of the larger background image on the homepage was being cut off due to the nature of the navigation bar. I decided that something needed to be done about that, thus another round was necessary.



Wireframes - Round 3

In the final round of the wireframing, there were several changes that were made, including the navigation bar, the services page, and the contact page and a mobile version was introduced.

The navigation bar was finalized and confirmed. The services page was changed for aesthetic purposes. A map was added to the contact page to give the user more information about the business' location. After this wireframe was approved by the client, a mobile wireframe was created.
